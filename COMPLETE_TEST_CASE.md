# RWA合规网关完整测试案例

## 测试目标

本测试案例旨在全面验证RWA合规网关的功能和安全性，确保系统在各种场景下都能正常运行，同时正确处理边界情况和异常情况。

## 测试环境

- **后端服务**：Go 1.26.1+
- **前端服务**：Node.js 18.0+
- **智能合约**：Solidity 0.8.20+
- **测试工具**：Node.js测试脚本
- **数据库**：SQLite（开发环境）

## 测试用例详细设计

### 1. 用户认证与授权

#### 1.1 用户注册

**测试用例1.1.1：正常注册**
- **输入**：有效的用户名、邮箱、密码、以太坊地址、角色
- **预期输出**：注册成功，返回成功响应
- **测试步骤**：
  1. 发送POST请求到 `/v1/auth/register`
  2. 验证响应状态码为200
  3. 验证响应包含成功信息

**测试用例1.1.2：重复用户名注册**
- **输入**：已存在的用户名、新邮箱、密码、以太坊地址、角色
- **预期输出**：注册失败，返回错误信息
- **测试步骤**：
  1. 先注册一个用户
  2. 使用相同的用户名再次注册
  3. 验证响应状态码为400
  4. 验证响应包含错误信息

**测试用例1.1.3：无效以太坊地址注册**
- **输入**：有效的用户名、邮箱、密码、无效的以太坊地址、角色
- **预期输出**：注册失败，返回错误信息
- **测试步骤**：
  1. 发送POST请求到 `/v1/auth/register`，使用无效的以太坊地址
  2. 验证响应状态码为400
  3. 验证响应包含错误信息

**测试用例1.1.4：密码长度不足注册**
- **输入**：有效的用户名、邮箱、密码（长度不足6位）、以太坊地址、角色
- **预期输出**：注册失败，返回错误信息
- **测试步骤**：
  1. 发送POST请求到 `/v1/auth/register`，使用长度不足6位的密码
